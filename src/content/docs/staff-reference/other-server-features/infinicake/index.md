Public-safe technical notes for staff who configure or support InfiniCake on 1MoreBlock.

> Verify destructive, economy-changing, permission-changing, and player-data commands against the installed build and an approved support case before running them.

## Commands

No plugin-owned staff command was verified for this feature. Manage it through its configuration or the owning platform documented below.

## Permissions

No stable plugin-owned permission nodes were verified. Confirm the installed build before adding grants.

## Placeholders

No plugin-owned placeholders were verified. PlaceholderAPI may still expose values through a separate expansion or an integrating plugin.

## Configuration and integrations

- Creation recipe and persistence behavior are controlled by the installed build and 1MoreBlock configuration.
- Protection plugins still control placement, breaking, and interaction.
- 1MB-CMIAPI BirthdayLanterns can optionally detect the InfinityCake/InfiniCake feature when present.

## Troubleshooting

- Confirm the item was created through the configured InfiniCake method.
- Test interaction outside protected regions before assuming the cake metadata is invalid.
- Preserve item metadata and placement steps when reporting a cake that becomes ordinary.
