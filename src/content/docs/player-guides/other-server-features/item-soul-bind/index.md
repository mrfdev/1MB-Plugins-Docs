title: "ItemSoulBind Guide"
description: "Bind selected items to a player so other players cannot use or keep them."
---

ItemSoulBind marks selected items for a specific owner. 1MoreBlock can use that ownership to protect special rewards, personal equipment, or other items that should not be freely transferred.

> Commands and features on 1MoreBlock depend on the current server configuration, world, and your permissions. A command listed here may be limited to certain ranks or contexts.

## What it adds

- Bind and unbind item-in-hand workflows.
- Bind an inventory, bind on use/equip/pickup, and group binding where configured.
- Owner checks when using, moving, dropping, or collecting bound items.
- Return and administrative recovery tools.

## Commands

| Command | What it does |
| --- | --- |
| `/bindItem` | Binds the held item to you when permitted. |
| `/unbindItem` | Unbinds the held item when permitted. |
| `/bindAll` | Binds eligible items in your inventory when available. |
| `/returnItems` | Returns recoverable bound items in supported flows. |

## Getting started

1. Hold the item that should be bound.
2. Use the command or menu supplied with that reward.
3. Read the resulting lore or message and test storage rules before relying on it.

## Player notes

- Bound items may be blocked from trading, dropping, containers, death transfer, or use by another player.
- A bound item is not automatically protected from every form of loss.
- Ask staff before attempting to rename, merge, repair, or transform a valuable bound item.
