This page introduces PlaceholderHealth as a public staff guide. It focuses on what the tool does, how authorized staff get started, and which commands are useful.

## What It Does

Read-only server health monitor for configured server and 1MB placeholders with ok/warn/error checks and Markdown exports.

## How Staff Use It

Start with /placeholderhealth. The sections below explain what PlaceholderHealth is for, which commands authorized staff can use, and what its output means.

Available features include:

- Parse configured PlaceholderAPI samples only.
- Support server-level samples and online-player-required samples.
- Flag unresolved placeholders, %...% output, errors, empty values, and slow parse times.
- Show ok/warn/error rows in console or in game for permitted admins.
- Export the latest or freshly-run check as a Discord-friendly Markdown report.
- Stay passive: it never edits PlaceholderAPI, CMI, holograms, configs, or playerdata.

## Quick Start

1. Start with `/placeholderhealth` to open or view PlaceholderHealth.
2. Use `/placeholderhealth info` when you want the in-game help or a short explanation.
3. Try `/placeholderhealth help` when you are ready to use one of the feature actions.
