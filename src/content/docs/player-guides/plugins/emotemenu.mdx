This page introduces Emotes from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

Emotes gives players a GUI for server emotes that are already implemented as server alias-editor commands, such as /cheer, /shrug, /freeze, /cozy, and similar custom aliases.

## How Players Use It

Start with /emotes. The sections below explain what Emotes is for, which commands players can try, and what to expect when your account has access.

Available features include:

- A searchable /emotes GUI for the server emote commands already available to the player.
- Clickable emote entries with plain descriptions and the exact command shape.
- Paginated chat lists for players who prefer commands over a GUI.
- Online-player pickers for emotes with optional or required targets.
- Safe handling for emotes that need non-player arguments.
- A Main Menu shortcut when the Menu plugin is installed and enabled.

## Quick Start

1. Start with `/emotes` to open or view Emotes.
2. Use `/emotes status` to check your current progress, settings, or state.
3. Use `/emotes help` when you want the in-game help or a short explanation.
4. Try `/emotes 1` when you are ready to use one of the feature actions.

- `/emotes`
- `/emotes 2`
- `/emotes search cheer`
- `/emotes search cozy`

## Good To Know

- If a command opens a menu, follow the buttons in-game; the website is only the learning guide.
