---
title: Emotes Commands
description: Player-friendly command guide for Emotes.
---

Emotes gives players a GUI for server emotes that are already implemented as CMI alias-editor commands, such as `/cheer`, `/shrug`, `/freeze`, `/cozy`, and similar custom aliases.

## Commands

<table>
  <thead>
    <tr>
      <th>Command</th>
      <th>What it does</th>
      <th>Example</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>/emotes</code></td>
      <td>Opens or shows the main Emotes player view.</td>
      <td><code>/emotes</code></td>
    </tr>
    <tr>
      <td><code>/emotes help</code></td>
      <td>Shows the Emotes commands available to you.</td>
      <td><code>/emotes help</code></td>
    </tr>
    <tr>
      <td><code>/emotes &lt;page&gt;</code></td>
      <td>Uses Emotes with the value you provide.</td>
      <td><code>/emotes 1</code></td>
    </tr>
    <tr>
      <td><code>/emotes search &lt;keyword&gt;</code></td>
      <td>Manages one of your Emotes entries.</td>
      <td><code>/emotes search cheer</code></td>
    </tr>
    <tr>
      <td><code>/emotes list [page]</code></td>
      <td>Lists available Emotes entries.</td>
      <td><code>/emotes list</code></td>
    </tr>
    <tr>
      <td><code>/emotes run &lt;emote&gt; [player]</code></td>
      <td>Runs the Emotes run action.</td>
      <td><code>/emotes run cheer</code></td>
    </tr>
    <tr>
      <td><code>/emotes status</code></td>
      <td>Shows your current Emotes status.</td>
      <td><code>/emotes status</code></td>
    </tr>
    <tr>
      <td><code>/emotes info</code></td>
      <td>Shows a friendly explanation of what Emotes does.</td>
      <td><code>/emotes info</code></td>
    </tr>
  </tbody>
</table>

## Examples

- `/emotes`
- `/emotes 2`
- `/emotes search cheer`
- `/emotes search cozy`
- `/emotes list`
- `/emotes list 2`
- `/emotes run cheer`
- `/emotes run cheer NikkiPixel`
- `/emotes run nomnom snacks`
- `/emotes run highfive Theo`
- `/emotes status`
- `/emotes info`
