This page introduces Emotes from a player point of view. It focuses on what you can do in-game, how to get started, and which commands are useful when the feature is available to you.

## What It Does

Emotes gives players a GUI for server emotes that are already implemented as server alias-editor commands, such as /cheer, /shrug, /freeze, /cozy, and similar custom aliases.

## How Players Use It

Start in-game with /emotes. The sections below explain what Emotes is for, which commands players can try, and what to expect when your account has access.

## Things You Can Do

- Opens a GUI with /emotes.
- Lists emotes in compact paginated chat rows with /emotes list [page].
- Item titles show the exact command shape, such as /cheer [player] or /shrug.
- Item lore explains what the emote does and whether a target is optional or required.
- Includes a bottom-left search button that opens a server text-input dialog, with a safe chat prompt fallback.
- Includes bottom-middle pagination controls.
- Includes a bottom-right barrier close button.
- Uses the shared light-blue GUI filler/border material so blocked slots stay visible across resource packs.

## Quick Start

1. Start with `/emotes` to open or view Emotes.
2. Use `/emotes status` to check your current progress, settings, or state.
3. Use `/emotes help` when you want the in-game help or a short explanation.
4. Try `/emotes 1` when you are ready to use one of the feature actions.

- `/emotes`
- `/emotes 2`
- `/emotes search cheer`
- `/emotes search cozy`

## Good To Know

- Some commands may be hidden until your group, event progress, or unlocks allow them.
- If a command opens a menu, follow the buttons in-game; the website is only the learning guide.
