This page introduces StartupDoctor from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

Startup/runtime diagnostics plus read-only folder doctor suggestions and export reports.

## How Players Use It

Start in-game with /startupdoctor status. The sections below explain what StartupDoctor is for, which commands players can try, and what to expect when your account has access.

## Things You Can Do

- Capture a startup report shortly after the plugin stack finishes enabling.
- Run on-demand diagnostics from command output.
- Show a compact summary for support triage.
- Verify server-API classes are visible to the feature jar.
- List registered 1MB CMIAPI features, categories, versions, and enabled state.
- Show suggested manual fixes for folder findings without deleting, moving, or editing files.
- Write the latest diagnostics report or folder-doctor report to the StartupDoctor cache folder for support/debug use.

## Quick Start

1. Start with `/startupdoctor status` to open or view StartupDoctor.
2. Use `/startupdoctor status` to check your current progress, settings, or state.
3. Try `/startupdoctor summary` when you are ready to use one of the feature actions.
