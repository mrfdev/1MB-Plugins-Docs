This page introduces FilterGuard from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

FilterGuard extends server chat-filter style checks to other player-written text surfaces: signs, wall signs, hanging signs, wall hanging signs, books, anvils, item names, and server nicknames.

## How Players Use It

Start in-game with /filterguard. The sections below explain what FilterGuard is for, which commands players can try, and what to expect when your account has access.

## Things You Can Do

- Checks sign text, wall sign text, hanging sign text, wall hanging sign text, book edits, anvil rename text, item-name rename text, and server nickname changes.
- Supports per-surface enabled toggles.
- Supports per-surface actions: log, alert, cancel, and cancel-and-alert.
- Keeps bounded recent detections for the current runtime.
- Provides a safe /filterguard test &lt;surface&gt; &lt;text&gt; command before enabling cancellation.

## Quick Start

1. Start with `/filterguard` to open or view FilterGuard.
2. Use `/filterguard status` to check your current progress, settings, or state.
3. Try `/filterguard recent` when you are ready to use one of the feature actions.
