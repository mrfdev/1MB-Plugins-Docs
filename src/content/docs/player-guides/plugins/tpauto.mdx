This page introduces TPAuto from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

TPAuto lets players automatically accept incoming teleport requests when the feature is available to them. It is useful when you are hosting, building with friends, or helping visitors and do not want to accept every request manually.

## How Players Use It

Start with /tpauto. The sections below explain what TPAuto is for, which commands players can try, and what to expect when your account has access.

Available features include:

- A personal toggle for automatically accepting incoming CMI /tpa and /tpahere requests.
- Trusted-role access so the feature can be limited to hosts, streamers, staff, or build teams.
- Separate on, off, toggle, reset, and status controls.
- Optional notifications for the requester and target after an automatic acceptance.
- CMI remains responsible for request validity, cooldowns, warmups, safety checks, and the teleport itself.

## Quick Start

1. Start with `/tpauto` to open or view TPAuto.
2. Use `/tpauto status` to check your current progress, settings, or state.
3. Use `/tpauto info` when you want the in-game help or a short explanation.
4. Try `/tpauto on` when you are ready to use one of the feature actions.

## Good To Know

- Some commands may be hidden until your group, event progress, or unlocks allow them.
- If a command opens a menu, follow the buttons in-game; the website is only the learning guide.
- Settings commands only change your own player preferences unless the page says otherwise.
