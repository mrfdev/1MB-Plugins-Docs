This page introduces MobHat from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

MobHat lets players wear a small whitelisted mob as a temporary cosmetic. It does not replace server's normal /hat item feature. Instead, it owns /mobhat and can optionally bridge only the specific /hat mob ... pattern into MobHat.

## How Players Use It

Start in-game with /mobhat. The sections below explain what MobHat is for, which commands players can try, and what to expect when your account has access.

## Things You Can Do

- Spawns a safe cosmetic copy of a whitelisted mob.
- Mounts the stable head cosmetic mob on the player with server/Bukkit passengers.
- Adds an experimental offset-position test bed for back, shoulder-left, and shoulder-right.
- Keeps the real targeted mob in place when using /mobhat target.
- Supports baby mobs for Ageable entities.
- Supports modern entity scale through the SCALE attribute where available.
- Can require one empty inventory slot before equipping, so it stays safe beside server /hat item workflows.
- Removes cosmetic mob hats on quit, death, world change, dismount, disable, and stale/orphan cleanup.

## Quick Start

1. Start with `/mobhat` to open or view MobHat.
2. Use `/mobhat status` to check your current progress, settings, or state.
3. Use `/mobhat help` when you want the in-game help or a short explanation.
4. Try `/mobhat info` when you are ready to use one of the feature actions.

## Examples

- `/mobhat info`
- `/mobhat help`
- `/mobhat list`
- `/mobhat preview frog`
- `/mobhat preview parrot`
- `/mobhat mob goat`
- `/mobhat mob chicken`
- `/mobhat position`
- `/mobhat position back`
- `/mobhat position shoulder-left`
- `/mobhat position reset`
- `/mobhat target`
- `/mobhat remove`
- `/hat mob goat`
- `/hat mob`

## Good To Know

- Some commands may be hidden until your group, event progress, or unlocks allow them.
- If a command opens a menu, follow the buttons in-game; the website is only the learning guide.
- Settings commands only change your own player preferences unless the page says otherwise.
