This page introduces Visit from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

Visit adds player-owned visit locations using hidden server warps. It behaves like a lightweight player-warp feature without exposing the generated warps in normal /warps lists.

## How Players Use It

Start with /visit. The sections below explain what Visit is for, which commands players can try, and what to expect when your account has access.

Available features include:

- One hidden personal visit warp that other players can use through /visit &lt;player&gt;.
- A public visit browser with owner messages and strict player-name lookup.
- Personal controls for visit visibility, offline visits, cooldowns, and optional money settings.
- Perked arrival titles, subtitles, particles, and sounds.
- A readable settings status page with clickable quick toggles.
- Persistent visit statistics and public top lists.
- Safe server-warp integration that keeps visit destinations out of normal warp lists.

## Quick Start

1. Start with `/visit` to open or view Visit.
2. Use `/visit info` when you want the in-game help or a short explanation.
3. Try `/visit set` when you are ready to use one of the feature actions.

## Good To Know

- Some commands may be hidden until your group, event progress, or unlocks allow them.
- If a command opens a menu, follow the buttons in-game; the website is only the learning guide.
- You may also see /pvisit as a shortcut.
- Top lists and leaderboards are public-facing progress views when the feature enables them.
- Settings commands only change your own player preferences unless the page says otherwise.
