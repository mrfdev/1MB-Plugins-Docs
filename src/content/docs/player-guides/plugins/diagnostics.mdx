This page introduces Diagnostics from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

Diagnostics is removable owner tooling for live server investigations.

## How Players Use It

Use this page as a friendly guide to Diagnostics. It explains what the feature is for, what players can try in-game, and which commands are available when your account has access.

## Things You Can Do

- Records player joins, quits, Bukkit kick events, server kick events, server AFK enter/leave/kick events, and enough context to tell whether a disconnect had a kick event or was just a generic quit.
- Tracks per-player recent command context, block movement, interactions, open inventory title, teleports, world changes, damage, AFK state, and join duration.
- Snapshots player state at quit: world, location, ping, address redacted by default, game mode, health, food, flight flags, sleep state, open inventory, server AFK state, and tracked AFK state.
- Adds packet-adjacent context when available: client brand, locale, client view distance, server protocol version, ViaVersion player/server protocol data, Floodgate Bedrock player data, Geyser connection data, and...
