This page introduces Forage from a public server-docs point of view. It focuses on what the feature does in-game, how to get started when you have access, and which commands are useful.

## What It Does

Forage is a player-facing nature progression plugin for /forage. It is built for server 26.1.2+ and Java 25 and depends on server, server library, and 1MB-CMIAPI-Lib.

## How Players Use It

The GUI uses the shared hardened GUI service with safe holders, cancelled clicks and drags, duplicate click protection, delayed close actions, and the global light-blue filler material. The v1 hub includes:

Forage now separates player progression XP from tool soul XP. Player XP and Forage Points are tuned for account progression, while tool XP is tuned for tool upgrades and soul growth.

## Things You Can Do

- Quest index with ready-to-claim, daily, weekly, and monthly pages
- Skill branch picker
- Read-only top boards for level, XP, points, actions, treasures, daily XP, and weekly XP
- Tool/family explanation
- Camp-only tool upgrades for held Forage tools, including tiers, supported diamond refinement, and controlled vanilla enchants
- Camp-only Repair &amp; Merge for worn Forage tools
- Camp turn-ins for plain vanilla forage items
- Multi-tier Forage Dust crafting and bounded growth pulses

## Quick Start

1. Start with `/forage` to open or view Forage.
2. Use `/forage status` to check your current progress, settings, or state.
3. Use `/forage info` when you want the in-game help or a short explanation.
4. Try `/forage help` when you are ready to use one of the feature actions.
