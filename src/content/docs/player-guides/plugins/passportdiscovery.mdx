This page introduces PassportDiscovery from a player point of view. It focuses on what you can do in-game, how to get started, and which commands are useful when the feature is available to you.

## What It Does

PassportDiscovery is the umbrella discovery-passport plugin for player-facing collection progress. It adds /passport and tracks stamps for worlds, warps, visits, biomes, blocks, items, consumables, tools, weapons, armor, kills, mounts, and sustained exploration in shared UUID playerdata.

## How Players Use It

Start in-game with /passport info. The sections below explain what PassportDiscovery is for, which commands players can try, and what to expect when your account has access.

## Things You Can Do

- Collection.gametypes is the allowlist. Gamemodes not listed do not collect stamps.
- Collection.vanish: false means server vanished players do not collect stamps.
- Collection.god: false means server god-mode players do not collect stamps.
- Collection.fly: false means server /fly, temp fly, or survival allow-flight players do not collect stamps. Elytra gliding still counts as normal survival play.
- Players can still view their passport while collection is paused.

## Quick Start

1. Start with `/passport info` to open or view PassportDiscovery.
2. Use `/passport status` to check your current progress, settings, or state.
3. Use `/passport info` when you want the in-game help or a short explanation.
4. Try `/passport stats oneblock` when you are ready to use one of the feature actions.

## Good To Know

- Some commands may be hidden until your group, event progress, or unlocks allow them.
- If a command opens a menu, follow the buttons in-game; the website is only the learning guide.
- Top lists and leaderboards are public-facing progress views when the feature enables them.
- Settings commands only change your own player preferences unless the page says otherwise.
