Commands can be permission-based. This list explains what common commands are meant to do when they are available to you.

<table class="command-table">
  <thead>
    <tr>
      <th>Command</th>
      <th>What it is for</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>/menu</code></td>
      <td>Open the main server menu and find common server features.</td>
    </tr>
    <tr>
      <td><code>/visit</code></td>
      <td>Manage your public visit spot and visit other players when available.</td>
    </tr>
    <tr>
      <td><code>/nick</code></td>
      <td>Pick a safer nickname style from the curated nickname picker.</td>
    </tr>
    <tr>
      <td><code>/birthday</code></td>
      <td>Manage birthday and server-anniversary lantern rewards.</td>
    </tr>
    <tr>
      <td><code>/recording</code></td>
      <td>Quiet selected messages while recording or streaming.</td>
    </tr>
    <tr>
      <td><code>/emotes</code></td>
      <td>Browse server emotes when the feature is available to you.</td>
    </tr>
    <tr>
      <td><code>/discordchat</code></td>
      <td>See Discord chat engagement and linked-account rewards when enabled.</td>
    </tr>
    <tr>
      <td><code>/votetokens</code></td>
      <td>Review vote-token reward trades and safe reward tools.</td>
    </tr>
  </tbody>
</table>

For the fuller introduction to each feature, open the [plugin player guides](../plugins/). Those pages explain what each feature does before listing commands.
