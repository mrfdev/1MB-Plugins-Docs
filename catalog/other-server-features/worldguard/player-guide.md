# WorldGuard

WorldGuard provides region-scale protection for server areas that are not managed solely by islands or plots. Regions can have owners, members, priorities, parents, and flags controlling what is allowed.

> Commands and features on 1MoreBlock depend on the current server configuration, world, and your permissions. A command listed here may be limited to certain ranks or contexts.

## What it adds

- Cuboid, polygonal, global, and nested regions.
- Owners, members, parent regions, priorities, and overlap resolution.
- Flags for build, interaction, PvP, entry, mobs, damage, messages, and more.
- WorldEdit selection integration and API hooks used by other plugins.

## Commands

This feature does not require a general player command. It appears automatically, through another menu, or through staff-created content.

## Getting started

1. Read entry/exit messages and signs in protected areas.
2. Use normal public interactions and routes.
3. Report a denial with exact coordinates, action, and the message shown.

## Player notes

- WorldGuard regions are separate from BentoBox islands, PlotSquared plots, and Bolt locks.
- Overlapping regions and priorities can make the effective rule different from one region's visible flag.
- Bypass access is privileged and should never be used for ordinary gameplay.
