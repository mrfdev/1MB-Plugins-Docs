# Multiverse-Inventories

Multiverse-Inventories controls which worlds share player inventory state. It lets 1MoreBlock separate game modes while allowing selected related worlds to share items, levels, health, or other data.

> Commands and features on 1MoreBlock depend on the current server configuration, world, and your permissions. A command listed here may be limited to certain ranks or contexts.

## What it adds

- World groups with shared or separated inventories.
- Configurable sharing of inventory, armor, ender chest, experience, health, hunger, and effects.
- Per-gamemode separation where configured.
- Import and conflict-management tools for administrators.

## Commands

This feature does not require a general player command. It appears automatically, through another menu, or through staff-created content.

## Getting started

1. Store important items before entering an unfamiliar game mode.
2. Expect inventory contents to switch when moving between separated world groups.
3. Return to the original world group before reporting an item as missing.

## Player notes

- An item visible in one world group may still exist safely in another.
- Game mode, armor, ender chest, potion effects, health, and XP can each have separate sharing rules.
- Do not move items between isolated modes through bugs or unintended plugin paths.
