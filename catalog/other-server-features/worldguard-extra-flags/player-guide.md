# WorldGuardExtraFlags

WorldGuardExtraFlags adds more flag types to WorldGuard. These extra controls can tune commands, movement, effects, entity behavior, item use, damage, and other region-specific behavior depending on the installed version.

> Commands and features on 1MoreBlock depend on the current server configuration, world, and your permissions. A command listed here may be limited to certain ranks or contexts.

## What it adds

- Additional flags registered into the normal WorldGuard flag system.
- Region-specific commands, effects, movement, interaction, and entity controls.
- Support for nuanced server areas without a separate protection platform.
- Normal WorldGuard priority, parent, overlap, and bypass behavior.

## Commands

This feature does not require a general player command. It appears automatically, through another menu, or through staff-created content.

## Getting started

1. Treat protected-area messages and denials as intentional region rules.
2. Move outside the region before assuming a feature is globally disabled.
3. Report the exact location, action, and denial/effect to staff.

## Player notes

- Extra flags are configured with WorldGuard `/rg flag`, not a separate player command.
- Available flags depend on the installed WorldGuardExtraFlags version.
- Overlapping WorldGuard regions still determine the effective result.
